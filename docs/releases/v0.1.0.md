# v0.1.0

**Initial Release** - April 2026

## Highlights

devspec is a Spec-Driven Development Evaluation CLI that evaluates project specifications against multiple SDD methodologies using LLM-as-a-Judge.

## Features

### SDD Type System

- Core type definitions with YAML frontmatter and markdown body
- Type inheritance via `extends` field
- Resolver loads types from project-local, user-global, or built-in sources
- Embedded definitions using `go:embed` for zero-configuration startup

### Built-in SDD Types

- **Kiro** (AWS): Three-phase workflow with EARS format acceptance criteria
  - Files: requirements.md, design.md, tasks.md
  - Spec directory: `.kiro/specs/`

- **SpecKit** (GitHub): Outcome-focused specifications with executable plans
  - Files: constitution.md (optional), spec.md, plan.md, tasks.md
  - Spec directory: `.speckit/`

- **PlexusOne**: Comprehensive enterprise specification methodology
  - Files: MRD.md (optional), PRD.md, TRD.md, PLAN.md, TASKS.md
  - Spec directory: `specs/`

### CLI Commands

- `check` - Detect SDD type in a directory with confidence scoring
- `validate` - Validate spec structure against SDD type requirements
- `init` - Scaffold spec files for a chosen SDD type
- `info` - Get detailed SDD type information for coding assistants
- `rubrics` - Get evaluation rubrics (GO/WARN/NO-GO criteria)
- `evaluate` - LLM-powered evaluation using LLM-as-a-Judge
- `version` - Display version information

### LLM Evaluation

- Two evaluation modes: standard (prompt-based) and tool-based
- Multi-provider support via OmniLLM: Anthropic, OpenAI, AWS Bedrock, Google Gemini
- Per-criterion scoring with weighted aggregation
- Status aggregation: NO-GO > WARN > GO
- Dry-run mode for testing prompts
- LLM-callable tools for codebase-aware evaluation

### Output Formats

- **TOON** (default): Token-efficient format, ~40% fewer tokens than JSON
- **JSON**: Pretty-printed with 2-space indent
- **JSON-compact**: Minified for APIs
- **Text**: Human-readable with status indicators
- **Markdown**: Documentation-ready with tables

### LLM-Callable Tools

Tools that LLMs can invoke during evaluation:

- `devspec_check` - Detect SDD type
- `devspec_validate` - Validate structure
- `devspec_info` - Get SDD type info
- `devspec_rubrics` - Get evaluation rubrics
- `read_file` - Read file contents (with 50KB limit and path security)
- `glob_files` - Find files by pattern (with path sandboxing)
